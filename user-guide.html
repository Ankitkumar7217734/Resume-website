üöÄ Getting Started

Opening the Editor

  1. Make sure the backend server is running: npm start
  2. Open index.html in your web browser
  3. You'll see a sample document in the editor

Your First Document

  1. Write: Type your LaTeX code in the left editor pane
  2. Compile: Click the green "Compile" button or press Ctrl+B (Windows/Linux) or Cmd+B (Mac)
  3. Preview: See your compiled PDF in the right pane
  4. Download: Click "PDF" or "DOCX" to download your document
üí° Tip: The editor automatically saves your work to browser storage. You won't lose your progress!

üé® Interface Overview

Header Bar

  • PDF Button: Downloads compiled PDF

Editor Pane (Left)

  • Professional code editor with LaTeX syntax highlighting
  • Line numbers for easy reference
  • Auto-indentation and bracket matching
  • Minimap for navigation (right side of editor)
  • Font size controls (+ and - buttons)

Preview Pane (Right)

  • Real-time PDF preview
  • Zoom controls (+ and - buttons)
  • Shows compilation errors if any

Status Bar (Bottom)

  • Shows current status (Ready, Compiling, Error)
  • Displays cursor position (line and column)

‚öôÔ∏è Settings

Auto-Compile

When enabled, automatically compiles your document after you stop typing.

  • Pros: See changes instantly
  • Cons: May slow down for large documents

Auto-Save

Automatically saves your work to browser's local storage.

üí° Recommended: Keep this enabled to prevent data loss!

Compile Delay

Sets the delay (in milliseconds) before auto-compilation triggers.

  • 2000ms (default): Good balance
  • Lower (500-1000ms): More responsive, more server load
  • Higher (3000-5000ms): Less server load, less responsive

üíæ Saving and Loading

Auto-Save

Your work is automatically saved to browser storage every second when you make changes.

Manual Save

Press Ctrl/Cmd + S to manually trigger a save.

Loading Previous Work

When you reopen the editor, your last document is automatically loaded.

‚ö†Ô∏è Important: Browser storage is limited to about 5-10MB and is cleared if you clear browser data. For important documents, always download a copy!

üì§ Exporting Documents

Export as PDF

  1. Compile your document first
  2. Click the "PDF" button in the toolbar
  3. Your browser will download document.pdf

Export as Word (.docx)

  1. Click the "DOCX" button (compilation happens automatically)
  2. Your browser will download document.docx

üîß Common LaTeX Commands

Document Structure

\documentclass{article}  % Document type
\begin{document}          % Start content
\end{document}            % End content

Sections

\section{Title}          % Main section
\subsection{Title}       % Subsection
\subsubsection{Title}    % Sub-subsection

Text Formatting

\textbf{bold text}
\textit{italic text}
\underline{underlined text}
\texttt{monospace text}

Lists

% Bulleted list
\begin{itemize}
    \item First item
    \item Second item
\end{itemize}

% Numbered list
\begin{enumerate}
    \item First item
    \item Second item
\end{enumerate}

Math

% Inline: $E = mc^2$

% Display:
\begin{equation}
    \int_a^b f(x) dx = F(b) - F(a)
\end{equation}

Tables

\begin{table}[h]
\begin{tabular}{|c|c|c|}
\hline
Header 1 & Header 2 & Header 3 \\
\hline
Cell 1 & Cell 2 & Cell 3 \\
\hline
\end{tabular}
\end{table}

‚ùå Troubleshooting

Compilation Errors

Error: "Missing $ inserted"

Cause: Math symbols used outside math mode

Solution: Wrap math in $...$ or \[...\]

Error: "Undefined control sequence"

Cause: Typo in command name or missing package

Solution: Check spelling or add required \usepackage{...}

Error: "File not found"

Cause: Missing image or include file

Solution: Remove the \includegraphics or add the file

Preview Issues

Blank Preview Pane

  • Click "Compile" button first
  • Check for compilation errors in error panel
  • Verify backend server is running

Connection Errors

  • Ensure npm start is running
  • Check that port 3000 is not blocked
  • Try restarting the server

üí° Tips and Best Practices

Writing Tips

  • Use comments: Add % comments to document your code
  • Structure first: Set up sections before writing content
  • Compile often: Catch errors early
  • Use packages: \usepackage{graphicx} for images, \usepackage{amsmath} for math
  • Label and reference: Use \label{} and \ref{} for cross-references

Performance Tips

  • Disable auto-compile for large documents
  • Increase compile delay if typing fast
  • Clear browser cache if editor becomes slow
  • Keep documents under 1000 lines when possible

üéì Example Workflows

Writing a Research Paper

  1. Start with "Article" template
  2. Add your title, author, and abstract
  3. Create sections: Introduction, Methods, Results, Discussion, Conclusion
  4. Add citations with BibTeX
  5. Include figures and tables
  6. Compile and download PDF

Creating a Resume

  1. Start with "Resume" template
  2. Update contact information
  3. Fill in education, experience, skills sections
  4. Adjust formatting to fit one or two pages
  5. Export as PDF for applications

Making Presentation Slides

  1. Start with "Beamer" template
  2. Update title slide information
  3. Add content slides with \begin{frame}
  4. Use blocks for emphasis
  5. Compile to see slide preview
  6. Download PDF for presenting
